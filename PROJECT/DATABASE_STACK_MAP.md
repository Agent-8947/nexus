# 🗄 КАРТА БАЗ ДАННЫХ: Экосистема Хранения NEXUS 2026

Это стратегический стандарт выбора хранилищ для AI-разработки.
**Принцип:** Выбирай инструмент под задачу, а не потому что это модно.

---

## 🗃 1. РЕЛЯЦИОННЫЕ БД — Структура и Связи

*Цель: Надежное хранение 80% данных проекта.*

| Инструмент | Когда использовать | Хостинг | Особенности |
| :--- | :--- | :--- | :--- |
| **Supabase** | Основная БД для SaaS / AI | Облако (Free Tier) | Postgres + Auth + Storage + pgvector. |
| **PostgreSQL** | Полный контроль над сервером | Railway / Render | Стандарт индустрии. Мощные расширения. |
| **SQLite** | Локальные инструменты (E:\) | Файл на диске | Один файл = вся БД. Идеал для Python. |
| **Cloudflare D1** | Edge-приложения | Cloudflare | SQLite на краю сети. Задержка < 10ms. |

---

## 📦 2. NoSQL — Скорость и Гибкость

*Цель: Кэширование, сессии и непредсказуемые структуры.*

| Инструмент | Когда использовать | Хостинг | Особенности |
| :--- | :--- | :--- | :--- |
| **Redis (Upstash)** | Кэш, очереди, Rate Limit | Serverless (Pay-per-req) | Ответ за миллисекунды. |
| **MongoDB Atlas** | Гибкая схема документов | Облако (Free Tier) | JSON-хранилище. Простая итерация схемы. |
| **PocketBase** | MVP за 15 минут | VPS / Local | Auth + Realtime + WebUI в одном бинарнике. |

---

## 🧠 3. ВЕКТОРНЫЕ БД — Память для ИИ (RAG)

*Цель: Поиск по смыслу (семантика) для контекста LLM.*

| Инструмент | Когда использовать | Особенности |
| :--- | :--- | :--- |
| **Supabase pgvector** | RAG внутри Postgres | Не нужна отдельная БД. Идеально для старта. |
| **Pinecone** | Огромные объемы векторов | Специализированная Managed Cloud БД. |
| **Chroma / Qdrant** | Локальная разработка / Docker | Open-source. Полный контроль над данными. |

---

## 🔍 4. ПОИСК — Текст и Семантика

*Цель: Когда SQL `LIKE` уже не справляется.*

| Инструмент | Когда использовать | Особенности |
| :--- | :--- | :--- |
| **Meilisearch** | Молниеносный поиск для SaaS | Настраивается за 10 мин. Поддержка RU. |
| **Typesense** | Поиск для сайтов / Документации | Опечаткоустойчивый, очень быстрый. |
| **Postgres FTS** | Базовый поиск в Supabase | Встроено. Бесплатно. Хватает для 90% задач. |

---

## 📁 5. ФАЙЛЫ — Объекты и Медиа

*Цель: Хранение документов (PDF, Docx) и изображений.*

| Инструмент | Когда использовать | Особенности |
| :--- | :--- | :--- |
| **Cloudflare R2** | Основное хранилище | S3-совместим. $0 за трафик (Egress). Дешево. |
| **Supabase Storage** | Файлы с Auth-защитой | Тесная интеграция с политиками БД (RLS). |
| **Локальная FS** | Десктоп на E:\ | Прямой доступ через Python `pathlib`. |

---

## 📊 6. АНАЛИТИКА — Большие Данные

*Цель: Анализ миллионов строк за секунды.*

| Инструмент | Когда использовать | Особенности |
| :--- | :--- | :--- |
| **ClickHouse** | Продакшн аналитика / Логи | Колоночная БД. В 100 раз быстрее Postgres. |
| **DuckDB** | Аналитика внутри Python | Анализ Parquet/CSV файлов без сервера. |

---

### 🔑 Главное правило NEXUS

**Supabase** закрывает 90% твоих потребностей (Postgres + Auth + Storage + pgvector). Добавляй другие БД (Redis, D1, R2) только тогда, когда есть конкретная техническая необходимость.
